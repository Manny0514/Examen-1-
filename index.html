Tema 2: Estructuración de una Página Web

Tema 2: Estructuración de una Página Web

Contenedor (wrapper):

El contenedor es un elemento que envuelve todos los demás componentes de la página. Generalmente, se utiliza para establecer un ancho máximo o mantener la estructura centralizada y ordenada en la pantalla. Proporciona un marco dentro del cual se colocan otros elementos como la cabecera, el contenido, el menú, etc.

Cabecera (header):

La cabecera suele ser la parte superior de la página y puede contener el logotipo del sitio, el título, eslóganes, enlaces de navegación principales o cualquier otra información importante que desee destacarse en la parte superior de la web.

Contenido (content):

El contenido es la parte central o principal de la página, donde se presenta la información específica o los servicios que ofrece el sitio web. Puede incluir texto, imágenes, videos y otros medios, dependiendo del propósito de la página.

Menú (menu):

El menú es el área destinada a la navegación del sitio web. Usualmente se presenta en forma de lista de enlaces a las diferentes secciones o páginas. Puede estar ubicado en la cabecera, el lateral o en cualquier parte que facilite la navegación para los usuarios.

Pie (footer):

El pie de página es la sección inferior de una página web. Su función es generalmente mostrar enlaces secundarios, información de contacto, derechos de autor o cualquier otro recurso relevante como políticas de privacidad o términos de servicio.

Lateral (sidebar):

El lateral es una sección que generalmente aparece a un lado del contenido principal. Se utiliza para presentar información adicional, como enlaces a artículos relacionados, publicidad, menús secundarios, o widgets como calendarios, búsqueda, etc.

Etiqueta div:

La etiqueta div (abreviatura de division) es un contenedor genérico que se utiliza en HTML para agrupar bloques de contenido o elementos de diseño. Permite estructurar el contenido sin añadir ningún estilo predeterminado, lo que hace que sea altamente personalizable mediante CSS.

Tema 3: Estructuración Semántica de una Página Web

Tema 3: Estructuración Semántica de una Página Web

La estructuración semántica de una página web utiliza etiquetas HTML que no solo definen la estructura visual, sino que también aportan significado al contenido para que sea más accesible tanto para los usuarios como para los motores de búsqueda. A continuación se describen los principales elementos semánticos que ayudan a estructurar el contenido de una manera más clara y lógica:

  • <header> (cabecera):

    Representa la sección de introducción o el encabezado de una página o sección del documento. Generalmente incluye logotipos, títulos, menús de navegación o enlaces relacionados. Se utiliza en el contexto de todo el documento o en secciones específicas.

  • <nav> (navegación):

    Indica una sección que contiene enlaces de navegación. Esta etiqueta se utiliza específicamente para agrupar el menú principal de la página o los enlaces a otras secciones importantes. Facilita la navegación dentro del sitio web.

  • <section> (sección):

    Agrupa contenido relacionado dentro de una página web. Cada sección puede tener su propio encabezado y contener varios tipos de información o contenido multimedia. Es útil para estructurar la página en partes bien definidas.

  • <article> (artículo):

    Representa una unidad de contenido autónoma y autocontenida, como un artículo de blog, una noticia o una publicación individual. Generalmente incluye encabezados, párrafos, imágenes y otros elementos que componen un bloque coherente de contenido.

  • <aside> (contenido complementario):

    Se utiliza para contenido relacionado de manera secundaria con el contenido principal, como anuncios, enlaces a recursos adicionales, barras laterales con widgets o información complementaria.

  • <footer> (pie de página):

    Representa el pie de una sección o de toda la página. Incluye información como derechos de autor, enlaces a términos y condiciones, políticas de privacidad, o cualquier otro contenido que suele aparecer al final de una página o sección.

  • <main> (contenido principal):

    Esta etiqueta se usa para identificar el contenido principal de una página, diferenciándolo del contenido repetido como menús, cabeceras o barras laterales.

  • <h1> - <h6> (encabezados):

    Encabezados que estructuran el contenido según su importancia. <h1> es el título principal, mientras que <h2> a <h6> representan subtítulos y niveles secundarios de encabezado.

  • <figure> y <figcaption> (figura y leyenda):

    La etiqueta <figure> contiene contenido gráfico, como imágenes, y <figcaption> proporciona una descripción o leyenda para esa figura.

  • <mark> (destacado):

    Utilizada para resaltar o destacar partes específicas del texto con importancia o relevancia especial.

  • <time> (tiempo):

    Define una fecha o intervalo de tiempo. Ayuda a identificar los datos relacionados con el tiempo, como la fecha de un artículo.

  • <address> (dirección):

    Se utiliza para mostrar información de contacto, como una dirección física o de correo electrónico.

Ventajas de la estructura semántica:

  • Mejora la accesibilidad: Los lectores de pantalla y las tecnologías de asistencia pueden interpretar mejor el contenido y su jerarquía.
  • Optimización para motores de búsqueda (SEO): Los motores de búsqueda valoran la estructura semántica para entender mejor el contenido de una página.
  • Mantenibilidad y claridad: El código es más comprensible y facilita la lectura y el mantenimiento por parte de los desarrolladores.
